Angular Service

發表於 2022-01-05 更新於 2024-01-06 分類於 Angular

前言

廣義的來說，服務可以是任何值、函式或是特性，
但通常來說，服務會定義成類別 ( class )，用來處理特定的事情和邏輯。
Angular將元件和服務區分開，
元件定義為處理畫面操作的角色，像是屬性綁定、事件綁定等等，
而像是向後端取得資料、檢核輸入或撰寫Log等等動作，
就會定義在服務裡面，再將服務注入到元件裡面使用。
如此一來，多個元件可以依賴同一個服務，
就能達到重複利用和模組化的效果，
另外也能讓元件輕量化，程式碼更整潔。

目錄

  • 依賴注入
  • 建立服務
  • 提供服務

依賴注入

Dependency Injection 是指依賴注入，簡稱就是 DI。
元件會根據需求，將所需要的服務注入到元件裡面，
而元件就能得到操控服務的權限。
以電商平台的例子來說，
產品元件和購物車元件，都會需要取得產品資訊，
這時候就能將取得產品資訊的邏輯定義在服務裡面。

建立服務

在Angular中，如果要將一個類別定義為服務，需要用到 @Injectable 裝飾符，
以便讓Angular將它視為依賴並注入到元件裡面。

  1. 注入器是主要的機制，Angular會在啟動專案時自動幫你建立，所以不需要自己建立注入器。
  2. 注入器會建立管理依賴的容器，讓它們可以被重複利用。
  3. 提供者會是一個物件，用來告訴注入器如何獲取或建立依賴。

以官方的Hero Service為例，
有一個Hero元件，是用來顯示英雄清單的資料，
而資料則是透過Hero Service所取得。
因此在元件被實體化時，須將此依賴注入到元件裡面。

這時候注入器就會在容器尋找是否已存在Hero Service這一個依賴，
如果存在，則將此依賴開放給元件使用；
反之，就會在容器裡面新增依賴並開放給元件使用。

提供服務

你必須為服務註冊一個提供者(Provider)，這樣服務才能被使用。

提供者可以被定義在服務的裝飾符裡面，
而 root 則表示此服務在整個專案裡面只會擁有一個，
每個元件透過注入來分享這一個服務。

備註: 倘若使用 AngularCLI 建立服務，就會自動設定為 root。

當然也可以將服務設定為特定模組所使用，

表示這個服務會被模組內的元件所共享。
又或者是特定元件，

表示這個服務在該元件內所共享。
總結上述，服務的提供者可以為

  • root - 根等級，表示整個專案
  • ngModule - 模組等級，表示單一模組內
  • Component - 元件等級，表示單一元件內

Angular Pipe

發表於 2022-01-05 更新於 2024-01-06 分類於 Angular

前言

我們可以使用 Pipe 來轉換字串、貨幣、日期或其他資料的顯示，
簡單來說，你可以將 pipe 視為一個function在Template裡面使用，
input → pipe → output
舉例來說，
當日期的資料為 西元 2021/05/10，
就可以透過 pipe 轉換為 民國 110/05/10。

目錄

  • 常用的 Pipe
  • Pipe 使用方式
  • 自定義 Pipe

常用的 Pipe

  • Date

    將日期轉換為本地格式

  • Upper Case

    英文字母轉換為大寫

  • Lower Case

    英文字母轉換為小寫

  • Currency

    將數字轉換為貨幣字串，格式也能在地化

  • Decimal

    將數字轉換為有小數點的字串，格式也能在地化

  • Percent

    將數字轉換為有百分比的字串，格式也能在地化

Pipe 使用方式

如果要使用 Pipe ， 就必須在Template使用 pipe operator ( | )

加上參數

Chain-Pipe

如果想要使用多個Pipe，就在Pipe串接其他Pipe。

自定義 Pipe

在某些時候，部分資料的轉換並沒有built-in Pipe可以使用，
那我們就可以透過客製化的Pipe來轉換資料，
而客製化Pipe在Template的使用方式也會如同built-in Pipe一樣。
舉例來說，當我取得學生的數學成績資料，發現成績都不甚理想，
故此想在畫面上多顯示加乘過後的成績。

  • 建立 Class

  • Implements Pipe Transform 介面

  • 如果為手動建立需在ngModule宣告Pipe，避免發生找不到類別的錯誤

  • 在Template裡面使用 Pipe 來轉換資料

Angular Directive

發表於 2022-01-05 更新於 2024-01-06 分類於 Angular

前言

指令是用來幫 HTML Element 附加上畫面邏輯和顯示的類別 (Class)，
舉例來說，當你的Template有一個日期欄位( input )，
使用者在輸入時，欄位會為20210511，
但如果能將欄位改為顯示2021/05/11，會更利於使用者操作和畫面優化。
藉由指令來為 input element 加上格式化的行為。
而Angular也提供了一系列的內建指令，協助我們開發表單、列表等等。

目錄

  • 指令種類
  • 屬性型指令
  • 結構型指令

指令種類

  • 元件
    元件可以視為指令的一種，同時也是最常見的一種指令。
  • 屬性型指令
    用來改變元素(Element)、元件或指令的行為或顯示。
  • 結構型指令
    新增或移除DOM的Elements來改變DOM的Layout。

屬性型指令

  • NgClass - 新增或移除 Css 的 Class，改變畫面顯示

    1. 使用字串


    2. 搭配方法(method)使用，並且傳入物件參數

      物件的Key為CSS Class的名稱，Value則使用Boolean來決定是否新增或移除。

  • NgStyle - 透過元件當前的狀態改變畫面的顯示，設定 inline style。

結構型指令

結構型指令主要是用來改變DOM的結構，
針對被附著的Element執行新增或移除之類的操作，藉而影響畫面上的顯示，
需要注意的是結構性指令的前面一定要有「 * 」，
這樣angular才能辨認它是結構型指令 。
舉例來說，當我們要向後端取得學生成績的資料時，
畫面上應該要顯示讀取的字樣或圖示，
而在資料回來以後，則不顯示讀取的字樣或圖示，
另外，資料回傳的格式會是陣列格式。

  • NgIf - 依條件新增或移除被附著的Element

    查詢中會顯示正在取得資料，讀取中；當查詢回來後，則顯示學生成績。

  • ngFor - 根據陣列產生重複的Element

  • ngSwitch - 依條件切換畫面的顯示
    以上述為例，假如畫面上想要分開顯示英數國的成績，
    那這時候就可以使用 ngSwitch ，他的用法就像程式碼的switch判斷式。
    ngSwitch 分別有三個指令

    1. ngSwitch - 屬性型指令，用來改變另外兩個指令的行為。
    2. ngSwitchCase - 結構型指令，當條件成立時，就會顯示相對應的畫面。
    3. ngSwitchDefault - 結構型指令，當ngSwitchCase都不成立時，就會顯示預設畫面。

    沒有選擇的情況下，就會顯示預設畫面。

    選擇一門科目後，ngSwitchCase條件成立，就會顯示相對應的畫面。

    特別注意的是，我們將 ngSwitchCase 附著在 ngContainer 上面，
    而 ngContainer 並不會在DOM出現，
    因此我們可以將結構性指令附著在上面，
    就不用再創造新的HTML元素，像是 div 之類的。

使用 @Input & @Ouput 實現元件互動

發表於 2022-01-05 更新於 2024-01-06 分類於 Angular

前言

在了解 @Input 和 @Output 這兩個裝飾符(decorator)如何建立元件互動前，
請先了解甚麼是屬性繫結和事件繫結。

目錄

  • @Input
  • @Output

@Input

所謂的 @Input decorator，
其實就是父元件透過屬性繫結的方式將資料傳送到子元件，
資料來源是父元件的屬性，資料接收者是子元件加上@Input 裝飾符的屬性。

  1. 子元件宣告一個屬性，並且在前面加上 @Input

  2. 父元件的Template放入子元件的selector

  3. 將父元件的屬性使用屬性繫結傳遞給子元件

@Output

所謂的 @Output decorator，
就是子元件透過事件綁定傳送資料給父元件，
資料來源是子元件自定義的事件(custom event)，資料接收者是父元件的方法。
使用 @Output 一定要搭配 EventEmiter 的型別使用，
EventEmitter透過emit()這個方法來觸發一個自定義的事件，
父元件就會接收事件，並做出回應。

  1. 在子元件有一個刪除按鈕，當使用者點擊 ‘刪除’，子元件會觸發自定義的刪除事件

  2. 在父元件的Template綁定事件，並使用 $event 取得子元件回傳的字串

  3. 在父元件的Component新增 處理刪除事件的方法

Angular Component

發表於 2022-01-05 更新於 2024-01-06 分類於 Angular

前言

元件可以說是Angular的基礎，也可以說是Angular專案裡最重要的一部分，
因為元件負責畫面的顯示和操作邏輯，
讓我們可以透過元件來展示資訊或從使用者身上取得資訊。

目錄

  • 元件介紹
  • 創建元件
  • Text Interpolation
  • Binding

元件介紹

元件是建立Angular專案的基礎之一，每一個元件都包含這四種要素，

  • HTML Template - 負責畫面的顯示

  • Typescript Class - 負責畫面的行為

  • CSS Selector - 被Template所使用的Tag，是用來告知Angular啟動此元件

  • CSS Styles - 裝飾畫面，像是調整文字的顏色、排版

創建元件

建立元件的方式有兩種，

  • 使用 Angular CLI 建立

    當在terminal執行此命令後，CLI會幫你建立元件的目錄和檔案，
    也會將此元件宣告到module的declarations。

    也可以使用簡寫後的指令去建立元件

    或是需要在指定目錄下建立元件

    • 資料夾結構

      • <component-name>
        • <component-name>.component.html
        • <component-name>.component.ts
        • <component-name>.component.css
        • <component-name>.component.spec.ts 測試檔

  • 手動建立

    1. 建立目錄 <component-name>

    2. 新增檔案 <component-name>.ts

    3. Import Angular的元件核心模組

    4. 加上元件的裝飾符並定義要在template被使用的Tag

    5. 定義Template

      有兩種方式，
      第一個是載入同個目錄的Html檔

      第二個就是直接將html定義在裡面

    6. 定義Styles

      Styles定義的方法和Template一樣，
      可以載入同個目錄下的CSS檔

      也可以直接寫在裡面

    7. 最後定義Class

Text Interpolation

動態改變畫面上的顯示，插入字串變數到HTML(Template)。

使用方式

  • 直接顯示在畫面上

    1. 在ts裡面宣告一個字串變數
    2. 在template 將變數塞在兩個大括號

  • 將變數 塞在 元素的attribute

  • 先做運算後再轉成字串顯示

  • 直接呼叫函式轉成字串顯示

Binding

Property Binding

使用屬性繫結可以設定元素、指令和子元件的屬性(properties)，
建立Component到Template的單向資料流。

  • 元素(HTML elements)
  • 指令(directives)
  • 子元件(components)

資料來源是元件的屬性，資料接收者則為下列

  • 元素(html element property)
  • 指令(class property)
  • 子元件(class property)

簡單來說，就是將元件的屬性傳送到指定元素的屬性上面，

補充

當屬性繫結和插值繫結都是以字串作為資料來源，那兩者可以達到同樣的效果。

但當資料來源型別不是字串(String)時，則一定要使用屬性繫結。

Event Binding

使用事件繫結可以讓元件接收使用者觸發的事件，
像是鍵盤按壓或滑鼠的移動、點擊和接觸，元件會對此事件做出回應。
舉例來說，當使用者點擊「儲存(指定事件)」按鈕，就會執行 onSave() 的 方法。

簡單來說，事件繫結就是建立Template到Component的資料流，
資料來源是使用者觸發的事件，接收者是元件的方法。

Two way Binding

雙向繫結其實就是結合屬性繫結和事件繫結，將資料流變成一個循環的圓圈。
